# Runes Capsule Webinar

**Meeting Date:** November 23, 2025, 4:00 PM

## Notes

### Runes Protocol and Technical Understanding

The meeting extensively covered how the Runes protocol integrates with Bitcoin's UTXO model to enable trustless token transfers, with Rapha Sierra providing a technical walkthrough (14:00).

Runes improves on BRC-20 by binding token ownership to Bitcoin UTXOs, creating a hybrid meta protocol that reduces reliance on third-party oracles (20:11).

- BRC-20 required external validation of token transfers, introducing trust risks.
- Runes encodes token amounts in Bitcoin transaction outputs, linking tokens to UTXOs, which Bitcoin natively protects via cryptographic signatures.
- This hybrid approach enhances security and trustlessness by leveraging Bitcoin's underlying blockchain mechanics.
- Rapha emphasized that while some external reading is still necessary, the protocol is more secure than previous meta protocols.

The protocol uses OP_RETURN outputs to encode token transfer data, associating asset amounts, identifiers, and transaction details directly on-chain (25:30).

- Rapha demonstrated decoding this data, showing how tokens like "Onboard Your Mom" are identified by block and transaction IDs.
- Transfer amounts and ownership are explicitly encoded, enabling clarity and verification.
- This precise encoding supports multi-asset management on Bitcoin without altering the base protocol.
- The approach facilitates seamless interaction with Stacks via clarity smart contracts.

Capsule acts as a multisig vault bridging Runes tokens trustlessly onto the Stacks blockchain, requiring user signatures for withdrawals (33:45).

- Deposits to Capsule lock tokens in a vault controlled by multiple signatures, ensuring sole user control.
- Rapha demonstrated a transaction sending 69,000 tokens into Capsule with a fee of approximately 5,000 sats, explaining inputs and outputs from Bitcoin and Taproot addresses (34:49).
- This mechanism eliminates middlemen risks common in conventional bridges, increasing user sovereignty.
- The multisig setup aligns with the Bitcoin philosophy of self-custody and security.

Rapha confirmed sending Runes tokens incurs no protocol fee inherently, but transaction fees apply as standard Bitcoin network fees, with optional fees added by bridges or protocols for operational costs (46:23).

- Bolu's question about excess fees was clarified: fees are network-level, not protocol-imposed.
- Adding fees is optional and context-dependent, especially when bridging tokens.
- This fee transparency supports user trust and cost predictability.
- The protocol's design avoids hidden charges, reinforcing openness.

### Technical Insights on Bitcoin and Token Mechanics

The session included foundational teaching on Bitcoin's UTXO model, digital signatures, and transaction flows, connected to Runes token operations (14:00).

Rapha explained Bitcoin's model where each coin is linked to a public key and ownership transfer requires a private key signature, ensuring secure chain of custody (16:27).

- This model forms the basis for Runes' token ownership and transfer mechanism.
- Understanding inputs and outputs in Bitcoin transactions was emphasized as critical to grasping token flows.
- The analogy of coins inside vaults (UTXOs) clarified user control and security.
- Bolu confirmed increased clarity on these technical concepts by meeting end.

The discussion covered how token transfers split and return change via UTXOs, ensuring accurate asset tracking and ownership integrity (29:50).

- Examples showed how sending 69,000 tokens deducts from a larger balance and returns change to the sender's address.
- This preserves Bitcoin's native transaction structure while enabling fungible token management.
- The hybrid protocol ensures that token data is embedded within Bitcoin transactions transparently.
- This understanding informs better wallet and protocol design decisions.

Stacks Clarity smart contracts can read Bitcoin UTXOs and OP_RETURN data to realize trustless cross-chain token bridging, a core innovation highlighted by Rapha (41:51).

- Clarity's ability to read Bitcoin data removes trust dependency on third parties.
- This integration supports Stacks' vision of extending Bitcoin's capabilities securely.
- Bolu recognized this as a superior approach compared to traditional bridges.
- The technology roadmap includes expanding such trustless interoperability for multi-chain assets.
